Introduction

DeckSmith is a powerful tool for automating the creation of decks, cards, and slides. Whether you're designing a Trading Card Game (TCG), a tarot deck, or business cards, DeckSmith allows you to define a layout once and generate hundreds of unique cards instantly.

Note: While the command-line interface supports JSON, the DeckSmith GUI is optimized for YAML 1.2, offering a more readable and concise syntax for your card definitions.

Installation

DeckSmith is built with Python. To get started, install it via pip:

pip install decksmith

Once installed, launch the graphical interface with:

decksmith --gui

Quick Start

The fastest way to learn is by doing. When you launch the GUI, you can start a new project which will create two essential files:

  • deck.yaml: Defines the visual layout of your cards.
  • deck.csv: Contains the data (text, image paths) for each card.

YAML Structure

Your deck.yaml file defines the dimensions of your cards and the elements placed on them. Here is a basic example:

card:
  width: 63.5
  height: 88.9
  units: mm
  margin: 0

elements:
  - type: text
    name: title
    content: "My Card Title"
    x: 10
    y: 10
    width: 43.5
    height: 10
    font_size: 12

Text Elements

Text elements allow you to place dynamic text on your cards. You can bind the content to your CSV data using the %column_name% syntax.

- type: text
  name: card_name
  content: "%Name%"  # Pulls from the 'Name' column in deck.csv
  x: 5
  y: 5
  width: 53.5
  height: 8
  font: "Arial"
  font_size: 14
  align: center
  color: "#000000"

Images

Images can be static (like a card frame) or dynamic (like character art). DeckSmith supports powerful transformations.

- type: image
  name: artwork
  path: "%ArtPath%"
  x: 5
  y: 15
  width: 53.5
  height: 40
  keep_ratio: true
  crop: true  # Crops the image to fit the dimensions

Shapes

You can draw rectangles, ellipses, and lines to add structural elements to your design.

- type: shape
  shape_type: rect
  x: 0
  y: 0
  width: 63.5
  height: 88.9
  fill: "#ffffff"
  stroke: "#000000"
  stroke_width: 2

Variables & CSV

The power of DeckSmith lies in its ability to merge data. Your deck.csv file should look like this:

Name,ArtPath,Description
Goblin,assets/goblin.png,A small but annoying creature.
Dragon,assets/dragon.png,A massive fire-breathing beast.

In your YAML, you reference these columns using percent signs, e.g., %Description%.

Exporting

Once your design is ready, use the Build button in the GUI to generate images. You can also export directly to a PDF for easy printing.
